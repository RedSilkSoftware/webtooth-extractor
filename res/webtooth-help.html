Workflow

Here I briefly explain the steps you need to take to get results out of a webpage. Basically you create one filter for every result-field that you wish to get in the result table after the extraction. You can use a combination of HTML tags, classes or attributes to search for it or construct a Regular Expression that browses through the bare HTML source code of the webpage. You setup all the filter you need and can extract all the data numerous times (until the structure of the webpage changes)!
Please also share your filters to publicly availabel webpages by uploading them to GitHub or your personal website and letting me know about them. It's not mandatory to do so though.

  1. Configuration & Identification of filters:

    First you should setup the configuration of this project including the webpage's URL that you want to extract data from in the properties. The webpage can also be a local HTML-File, load it with the anchor-icon.

  2. Create new filters:

    Click into the "Label of new filter" field in the "Filter List"-Dock-Window and enter as many filters with the correct syntax as you wish. You can also remove them here with the syntax below. Pressing [Enter] executes your command, no need to click the "Add new filter / Remove" button on the side. Each filtername must be unique!

    Syntax: filterlabel ; command # amount

    • Filterlabel or -name:

      Any normal text is allowed here. Best is to name it according to the resulting field's name that you wish to get from it.

    • Command

      The commands are either R for a Regular-Expression filter, T for a HTML-Tag filter, D to delete a filter with the same name or D* to delete several filters that match the name with the added wildcard *. The commands can be entered in capitals or lower caps.

    • Amount

      For the R and T command you can optionally add a # followed by a number to added many filters of the same type at once, they will be labelled with increasing numbers.

    • Examples

      opening;t ⇰ A filter with the name "opening" of type HTML-Tag is created
      totalamount;r ⇰ A filter with the name "totalamount" of type Regular-Expression is created
      currency;t#5 ⇰ Five filters with the names currency1 to currency5 of type HTML-Tag are created
      totalamount;d ⇰ The filter with the name totalamount is removed/deleted
      currentDate;r ⇰ A filter with the name "currentDate" of type Regular-Expression is created
      curren*;d* ⇰ All filters that begin with "curren" are removed, this includes the currency and currentDate filters too, so be extra careful with this option and save the project before.

  3. Define Filter Criterias:

    The Regular-Expression filter is the most straight-forward of the two types, just examine the webpage's HTML source code with the Web-Inspector and then craft a RegEx that matches best using regular Perl syntax.
    The HTML filter offers you several options to combine. The more unique a field is the easier it is to find. If you only have 1 HTML-Tag of type "H5" then just enter this into the tag field and voil√†. But usually webpages are not constructed to be easily extractable so you will often find several fields with the same tag, same class, attribute or ID. Like with a Google search, finding the right combination of search criterias is the key to success. Maybe the field you need is unique in the combination of a "A" tag and a "currency" class. In some cases even all the atrtributes won't help you extract the right field, then you can look at the parent tag to the one you need. Let's say that your "A" tag is inside a "DIV" tag (the parent) and has the class name of "trader" and with those two additional criterias you are now able to get the data you wanted.

  4. Fish (Scan) for results

    Everytime you defined a new filter you should test it and run the Scan the webpage now which runs through all filters and outputs the found results in the table of the Results-Dock-Window. Every scan will also save a screenshot of that webpage as it is in this moment, it is saved with the name of the project-file in the same folder/directory. You can use it later as a reference or just to check if the content/design changed.

  5. Save the results (while you still can)

    The easiest way to save the results after a scan of the webpage is to click on the anchor-icon which will copy it into the clipboard in a structured manner specially suitable for spreadsheets.
    (Not yet implemented) You can also export the results as a CSV (comma-separated values) or a JSON file for later use into your database for example.

Menus

Here the menus are briefly explained, they should mostly be self-explanatory though, else the GUI needs tuning.

  • File

    • New Project (Ctrl-N): Creates a new project saving or discarding the existing data at your request.

    • Open Project (Ctrl-O): Opens an existing and previously saved WebTooth project file, usually with the extension .wpjson.

    • Save Project (Ctrl-S): Saves your data to a new project file or the currently active one.

    • Export Data...: Allows you to export the resulting table you extracted as a CSV or JSON file. If you don't need that, you can also copy the table into the clipboard and paste it into your favorite spreadsheet application.

    • Properties (Ctrl-P): Opens the properties dialog where you can define certain settings for this project.

    • Quit Application (Ctrl-Q): Exits the application and prompts you to save your data if unsaved.

  • Browser

    • Set Zoom Factor...: Enlarge or shrink the browsers view with the zoom levels of 50% (Ctrl-0), 100% (Ctrl-1), 125% (Ctrl-2), 150% (Ctrl-3)

    • Set Content As Editable: A checkable option that allows you to edit Text directly in the browser window. Off by default.

    • Show Web Inspector Dock: Checkable option to display the Web-Inspector dock-window which lets you find the HTML tags you need to define the extraction-filters. On by default.

  • Window

    • Show Filter List Dock: Checkable option to show or hide/close this dock-window. On by default.

    • Show Extraction Properties Dock: Checkable option to show or hide/close this dock-window. On by default.

    • Show Results Panel Dock: Checkable option to show or hide/close this dock-window. On by default.

  • Help

    • Help (F1): Opens this help page in the browser view.

    • About (Ctrl-A) Opens the About dialog window that displays info about this application, including the current version, some system informations and web-links to the webpages of products (right-click the links to copy it).
